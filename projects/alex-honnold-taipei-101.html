Taipei 101 × Alex Honnold — Interactive Climb Simulator

A cinematic, zero-dependency interactive web experience simulating Alex Honnold's historic free-climb of Taipei 101.

Interactive Simulation
Taipei 101 skyline with a live climb replay and telemetry dashboard
Built as a single static page, the simulator combines SVG animation, time controls, simulated biometrics, theme switching, achievements, and shareable state without any frameworks or build tooling.

Overview

Built a fully client-side interactive simulation of Alex Honnold's January 25, 2026 free-climb of Taipei 101 — 101 floors, 508 m (1,667 ft) of vertical glass and steel. The experience features a real-time SVG climber ascending a stylized Taipei skyline at Honnold's actual pace, with live telemetry, simulated biometrics, achievement unlocks, day/night theming, and shareable state — all in a single HTML file with zero dependencies.

Key Features

  • Real-time climb simulation at Alex Honnold's actual pace of 0.093 m/s
  • Playback speed control from 1× to 360× with a scrub slider
  • Three difficulty modes: Standard, Speed, and Endurance
  • Simulated biometrics for heart rate, O2 saturation, and climbing power
  • Achievement unlocks with toast notifications at Floor 25, Floor 50, Floor 75, and the summit
  • Day/night theming with animated clouds, twinkling city lights, and a sun/moon swap
  • Dynamic atmosphere effects at high altitude plus a summit particle celebration
  • Keyboard shortcuts for play/pause, reset, and theme control
  • Shareable URL state encoding speed, time, and difficulty
  • Responsive layout with single-column behavior below 1100px and accessibility support through aria-live regions, semantic HTML, and prefers-reduced-motion

Role and Stack

Role: Solo developer — design, animation, and implementation.

Tech Stack: HTML, CSS, JavaScript, SVG (vanilla — no frameworks, no build tools).

Tags: Interactive, Data Visualization, Animation, SVG, Front-End, Simulation.
